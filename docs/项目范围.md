# TechMind 项目范围

本文档记录项目模块边界。项目已完成论坛业务、异步任务、AI 内容理解、可观测底座、告警中心、SRE 诊断报告、React 前端和 kind K8s 部署；当前实现同时包含管理员服务端鉴权、受 Bearer Token 保护的 Alertmanager Webhook、Prometheus 指标证据采集，以及 Worker 9091 metrics 暴露。

## 模块边界

- `cmd/server`: Go + Gin API Server 入口。
- `cmd/worker`: Redis Stream 异步任务 Worker 入口，并在 9091 暴露 Prometheus metrics。
- `internal/controller`: HTTP 协议层。
- `internal/logic`: 业务编排层。
- `internal/dao`: MySQL、Redis、Milvus 数据访问层。
- `internal/monitor`: 指标、慢请求和错误事件相关能力。
- `internal/alert`: Alertmanager 接收、去重、增强和关联。
- `internal/agent`: Skill、Prometheus/数据库/Redis MCP、RAG 和诊断报告。
- `internal/ai`: 模型、Embedding 和 Prompt 封装。
- `internal/worker`: 异步任务消费、重试和死信处理。
- `internal/middleware`: API Server 中间件。
- `internal/pkg`: 项目内部通用包。
- `frontend/`: React 19 + Vite + TypeScript 前端，论坛（浅色）+ 管理后台（深色）双主题 SPA。
- `deploy/kind/`: kind 集群一键部署（MySQL/Redis/Prometheus/Alertmanager/Metrics Server + Helm 应用）。
- `deploy/helm/techmind/`: Helm Chart，含 server/worker/frontend 三个 Deployment、Worker metrics Service 和 Secret 注入。
- `deploy/docker/`: Docker Compose 全栈编排。

## 当前边界与后续项

- Agent 当前会读取 Prometheus、MySQL、Redis、告警、部署变更和 Runbook；Kubernetes Event/Pod 与 Milvus 健康检查仍是后续扩展。
- kind 环境默认不部署 Milvus，RAG 会降级；要验证向量检索需单独提供 Milvus。
- 二期可选扩展：OpenTelemetry + Jaeger 链路追踪、Loki 日志系统、持久化 Prometheus/Grafana 存储、ArgoCD 变更记录、K8s client-go watch Deployment 自动记录变更。
